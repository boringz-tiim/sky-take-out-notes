# 项目结构
1. sky-take-out maven父工程，统一管理依赖版本，聚合其他子工程
2. sky-common 子模块，存放公共类、工具类等
3. sky-pojo 子模块，存放实体类、VO、DTO
4. sky-server 子模块，后端服务，存放配置文件
-- sky-pojo
**Entity** 实体，通常和数据库中的表对应
**DTO** 数据传输对象，通常用于程序中各层之间传递数据
**VO** 视图对象，为前端展示数据提供的对象
**POJO** 简单JAVA对象，只有属性和对应的getters/setters方法，不包含业务逻辑

# 后端环境搭建-nginx反向代理
 1. 提高访问速度
 2. 进行负载均衡-把大量的请求按照我们指定的方式均衡的分配给集群中的每台服务器
 3. 保证后端服务的安全
## nginx 反向代理的配置方式：
**nginx.conf**
```
server{
    listen 80;
    server_name localhost;
    location /api/ {
        proxy_pass http://localhost:8080/admin/;
    }
}
https://localhost/api/employee/login ->
http://localhost:8080/admin/employee/login
```
## nginx 负载均衡的配置方式
**nginx.conf**
```
upstream webservers {
    server 192.168.100.128:8080;
    server 192.168.100.129:8080;
}

server {
    listen 80;
    server_name localhost;

    location /api/ {
        //转发地址webservers 后端服务ip和端口
        proxy_pass http://webservers/admin/;
    }
}
```
### 负载均衡策略
- 轮询（默认）：每个请求按时间顺序逐一分配到不同的后端服务器，如果后端服务器down掉，能自动剔除。
- 加权轮询weight：根据服务器的响应时间来给予其不同的权重，响应时间短的服务器获得更高的权重。
- 最少连接：保证每个后端服务器都得到足够的请求，响应时间最短的服务器获得更多的请求。
- IP Hash：根据客户端的IP地址进行hash，相同IP地址的请求都分配到同一台后端服务器。

## API相关注解
@Api (在类上)：负责归类，决定这一组接口叫什么名字。

@ApiOperation (在方法上)：负责解释，具体说明单个接口是做什么的。

@ApiImplicitParams (在方法上)：用于描述请求参数，包括参数类型、参数名称、是否必填、示例值、参数描述等。

## DTO VO
DTO：主要用于 Service 层 和 Controller 层。当后端需要接收前端传来的数据，或者需要给前端返回数据时，通常会使用 DTO。它也常用于微服务之间远程调用（RPC）时传递数据。
VO：主要用于 Controller 层，专门用来响应前端的请求，返回给前端用于页面展示。
