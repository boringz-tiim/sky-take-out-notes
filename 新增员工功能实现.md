# BeanUtils.copyProperties(employeeDTO, employee)
**将源对象第一个参数的属性值，复制到目标对象第二个参数中**
它会自动匹配两个对象中 属性名相同 的字段，把 employeeDTO 里的值（如 username、phone）复制到 employee 对象对应的字段里。
注意：它执行的是 浅拷贝，并且如果属性名对不上或者类型不一致，就不会复制（也不会报错，只是忽略）。

# Swagger 详解

## 一、Swagger 是什么？

Swagger 是一个用于生成、描述和调用 RESTful API 接口的工具，解决了前后端开发中接口文档的编写与同步问题。

### 核心价值

- **文档与代码绑定**：通过在 Java 代码中添加注解，Swagger 自动生成实时更新的可视化文档
- **可交互**：生成的文档页面支持在线测试接口
- **零维护成本**：代码变更时文档自动同步

### 访问地址

启动项目后访问：`http://localhost:8080/swagger-ui/index.html`

---

## 二、Swagger 常用注解

| 注解 | 使用位置 | 作用 | 示例 |
| :--- | :--- | :--- | :--- |
| `@Api` | Controller 类上 | 给整个类打标签，用于分组 | `@Api(tags = "员工相关接口")` |
| `@ApiOperation` | Controller 方法上 | 解释接口的作用 | `@ApiOperation(value = "查询员工", notes = "根据ID查询")` |
| `@ApiParam` | 方法参数上 | 解释单个参数的含义 | `@ApiParam(value = "员工ID", required = true) Long id` |
| `@ApiModel` | DTO/VO 类上 | 对数据对象进行说明 | `@ApiModel(value = "员工登录请求体")` |
| `@ApiModelProperty` | DTO/VO 的属性上 | 解释字段的含义 | `@ApiModelProperty(value = "用户名", example = "张三")` |
| `@ApiImplicitParam` | Controller 方法上 | 描述非对象类型的参数 | `@ApiImplicitParam(name = "id", value = "用户ID", paramType = "path")` |

```java
/**
     *
     * 处理SQL异常
     * @param ex
     * @return
     */
    @ExceptionHandler
    public Result exceptionHandler(SQLIntegrityConstraintViolationException ex){
        //Duplicate entry 'zhangsan' for key 'employee.idx_username'
        String message=ex.getMessage();
        if(message.contains("Duplicate entry")){
            //动态提取字符串
            String[] split=message.split(" ");
           String username= split[2];
           String msg=username+ MessageConstant.ALREADY_EXISTS;
           return Result.error(msg);
        }else{
            return Result.error(MessageConstant.UNKNOWN_ERROR);
        }
    }
```
